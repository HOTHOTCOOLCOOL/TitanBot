# 知识库系统发展路线图

## 📋 当前状态总结

### 已完成的功能 (MVP)

#### 1. 记忆整合重构 ✅
- **空闲触发**：10分钟空闲后自动触发（只触发一次）
- **用户确认触发**：每20个对话询问用户确认
- **后台 Idle Checker**：每30秒检查session文件
- **会话归档**：旧消息归档到 HISTORY.md

#### 2. 任务知识库 (Task Knowledge) ✅
- **存储结构**：`memory/tasks.json`
- **功能**：
  - save: 保存任务（需用户确认）
  - search: 搜索任务
  - get: 获取详情
  - list: 列出所有
  - delete: 删除
- **规则**：知识库只是缓存，必须先用工具获取最新数据

#### 3. 工具优化 ✅
- System prompt 明确区分 message vs outlook.send_email
- Save Skill 功能已实现

## 🔮 未来规划

### Phase 1: 增强版任务知识库（近期）

```
目标：让知识库更加智能和实用
```

1. **任务参数结构化**
   - 当前：params 是原始 dict
   - 改进：定义标准化的任务模板（TaskTemplate）
   ```python
   class TaskTemplate:
       name: str
       description: str
       required_params: list[str]
       optional_params: dict[str, Any]
       steps: list[Step]
       output_schema: dict
   ```

2. **自动学习**
   - 从成功任务中提取模式
   - 自动生成任务模板
   - 相似任务推荐

3. **知识图谱**
   - 任务之间的关系
   - 参数依赖图
   - 执行历史追溯

### Phase 2: 多模态知识库（中期）

```
目标：支持更丰富的知识类型
```

1. **邮件知识库**
   - 发件人画像
   - 邮件模式识别
   - 自动分类规则

2. **联系人知识库**
   - 用户关系图谱
   - 沟通历史摘要
   - 偏好设置

3. **文档知识库**
   - 项目文档索引
   - 代码片段收藏
   - 会议记录

### Phase 3: 向量化知识库（部分完成）

```
目标：语义搜索和智能推理
```

1. **向量嵌入** ✅
   - 使用 embedding model（已集成原生本地 `sentence-transformers/all-MiniLM-L6-v2`）
   - 语义相似度搜索（已使用 `ChromaDB` 索引化）
   - 自然语言查询

2. **RAG 增强** ✅
   - 结合 LLM 进行知识推理（已整合进入 `VectorMemory` 进行搜索辅助）
   - 自动知识更新（`_consolidate_memory` 自动维护向量数据库状态）
   - 跨知识库关联

3. **个性化学习**
   - 用户习惯建模
   - 自适应工作流
   - 预测性建议

## 📊 技术债务

1. **测试覆盖不足** - 关键功能需要单元测试
2. **错误处理** - 需要更健壮的异常处理
3. **日志优化** - 调试日志需要更清晰
4. **配置管理** - 硬编码的配置需要外部化

---

## 💡 核心原则

1. **知识库是辅助，不是替代** - 始终获取最新数据
2. **用户确认后才能学习** - 避免错误知识污染
3. **渐进式增强** - 从简单开始，逐步复杂化
4. **可解释性** - 每个知识条目可追溯来源
