# Nanobot 记忆系统问题分析与改进方案

## 问题描述
用户报告：nanobot能完成复杂任务（如读取Outlook邮件、分析附件、发送邮件），但无法重用这些学习到的能力。表现为一次性完成任务，但后续无法自动复用相同的工作流。

## 当前记忆系统分析

### 1. 现有记忆机制
- **MEMORY.md**：长期记忆，通过`_consolidate_memory()`更新
- **HISTORY.md**：历史记录，通过grep搜索
- **技能系统**：`workspace/skills/`目录下的SKILL.md文件

### 2. 记忆整合流程
```python
# 在AgentLoop._consolidate_memory()中：
1. 当会话消息超过memory_window(默认50)时触发
2. 将旧消息格式化为文本
3. 调用LLM提取：
   - "history_entry": 摘要（追加到HISTORY.md）
   - "memory_update": 长期记忆更新（覆盖MEMORY.md）
4. 仅更新最近`len(session.messages) - memory_window//2`条消息
```

### 3. 技能加载机制
```python
# 在ContextBuilder中：
1. 始终加载的技能(always_skills)：直接包含在系统提示中
2. 可用技能：显示摘要，agent需用read_file读取SKILL.md
3. 技能存储在workspace/skills/{name}/SKILL.md
```

## 识别出的核心问题

### 问题1：记忆与技能分离
- **记忆**：存储事实、偏好、上下文
- **技能**：存储可执行的工作流
- **缺失**：完成任务后自动创建技能的机制

### 问题2：记忆检索效率低
- MEMORY.md作为系统提示完整包含 → 上下文浪费
- HISTORY.md需要主动grep → 依赖agent主动查询
- 无智能检索：基于语义的相关性检索

### 问题3：整合触发条件不合理
- 仅当消息超过50条才触发 → 重要早期对话可能被忽略
- 无重要性检测 → 所有对话同等对待
- 无实时学习 → 需要积累大量对话

### 问题4：技能发现困难
- agent需要知道技能存在才能使用read_file读取
- 无技能推荐机制
- 技能元数据不完善

## 改进方案

### 方案1：增强记忆检索（短期）
1. **向量化记忆存储**：使用本地向量数据库（FAISS/Chroma）
2. **语义检索**：基于查询相似度检索相关记忆
3. **混合检索**：结合关键词(grep)和语义搜索

### 方案2：自动技能提取（中期）
1. **工作流记录**：记录工具调用序列和参数
2. **技能模板生成**：将成功的工作流转为SKILL.md
3. **技能优化**：压缩、泛化工作流步骤

### 方案3：改进整合触发（短期）
1. **重要性评分**：基于工具使用、用户反馈等评分
2. **即时学习**：重要对话后立即整合
3. **增量更新**：实时更新向量索引

### 方案4：技能管理系统（长期）
1. **技能目录**：带分类和标签的技能库
2. **技能推荐**：基于当前任务推荐相关技能
3. **技能版本管理**：支持技能迭代更新

## 实施优先级

### 第一阶段（1-2周）：基础改进
1. 实现重要性评分和即时记忆整合
2. 增强记忆提示策略（动态检索而非完整包含）
3. 添加技能创建工具

### 第二阶段（2-4周）：高级功能
1. 实现向量化记忆存储和检索
2. 自动技能提取和工作流记录
3. 改进技能发现和推荐

### 第三阶段（4-8周）：完整系统
1. 完整的技能管理系统
2. 工作流可视化编辑
3. 技能性能监控和优化

## 技术选择

### 向量存储选项
1. **FAISS**（Facebook AI Similarity Search）
   - 优点：高性能，纯Python/C++，无外部依赖
   - 缺点：需要手动管理序列化

2. **Chroma**（开源向量数据库）
   - 优点：功能完整，API友好
   - 缺点：额外依赖，可能较重

3. **SQLite + 向量扩展**
   - 优点：轻量，与现有系统兼容
   - 缺点：需要实现相似性搜索

### 嵌入模型选项
1. **all-MiniLM-L6-v2**（sentence-transformers）
   - 优点：轻量（80MB），性能好
   - 缺点：需要sentence-transformers包

2. **本地BERT模型**
   - 优点：完全本地，可定制
   - 缺点：资源消耗大

3. **OpenAI embeddings**（通过API）
   - 优点：质量高，简单
   - 缺点：需要网络，有成本

## 预期效果

### 用户体验改进
1. **更好的记忆重用**：agent能回忆起相关任务和工作流
2. **自动技能创建**：成功完成任务后自动保存为技能
3. **智能推荐**：基于当前任务推荐相关技能和记忆

### 性能指标
1. **记忆检索准确率**：从<30%提升到>80%
2. **技能重用率**：从接近0提升到>50%
3. **用户满意度**：显著减少重复指令

## 风险评估与缓解

### 技术风险
1. **向量数据库复杂度** → 从简单SQLite方案开始
2. **嵌入模型性能** → 选择轻量级模型
3. **技能泛化难度** → 从手动技能创建开始

### 兼容性风险
1. **向后兼容** → 保持现有文件格式
2. **配置迁移** → 提供自动迁移工具
3. **性能影响** → 逐步引入，可配置启用

## 具体实施计划

### 第1步：创建技能提取工具
在`nanobot/agent/tools/`中添加`save_skill_tool.py`，允许agent将当前工作流保存为技能。

### 第2步：改进记忆提示
修改`ContextBuilder.build_system_prompt()`，动态检索相关记忆而非完整包含。

### 第3步：实现重要性评分
在`AgentLoop._process_message()`中添加重要性评分，重要对话立即触发记忆整合。

### 第4步：向量化记忆存储（可选）
实现基于SQLite的简单向量存储，支持语义检索。
